1. Overview & Introduction

Microeconometric Analysis คืออะไร?

Microeconometric Analysis คืออะไร?

Microeconometrics คือการวิเคราะห์ข้อมูลระดับบุคคล (individual-level data) เกี่ยวกับพฤติกรรมทางเศรษฐกิจ

ต่างจาก macroeconometrics ที่ใช้ข้อมูลรวม (aggregate data เช่น GDP, CPI)

ตัวอย่างข้อมูล:

  • ค่าใช้จ่ายสุขภาพรายบุคคล (MEPS)
  • การตัดสินใจเข้าร่วมตลาดแรงงาน
  • ความต้องการซื้อประกัน
  • จำนวนครั้งที่ไปพบแพทย์

ทำไมต้องศึกษาระดับจุลภาค?

ข้อมูลรวมซ่อน ความหลากหลาย (heterogeneity) ระหว่างบุคคลไว้

ผลกระทบของนโยบายมักแตกต่างกันตามลักษณะของแต่ละคน — ข้อมูลระดับจุลภาคช่วยให้เห็นความแตกต่างนี้

ข้อมูลที่ใช้บ่อย

  • Cross-section: สุ่มตัวอย่าง ณ จุดเวลาเดียว
  • Panel data: ติดตามบุคคลเดิมหลายช่วงเวลา
  • Repeated cross-section: สุ่มใหม่ทุกรอบ

ลักษณะเด่นของ Microeconometric Data

Discreteness & Nonlinearity

ตัวแปรตามมักไม่ต่อเนื่อง เช่น ซื้อ/ไม่ซื้อ (binary), จำนวนครั้งไปหาหมอ (count) → ต้องใช้โมเดลที่เหมาะสม

Greater Realism

สร้างโมเดลจากทฤษฎีเศรษฐศาสตร์จุลภาคโดยตรง เช่น utility maximization, labor supply → ความสอดคล้องระหว่างทฤษฎีกับแบบจำลองเชิงประจักษ์

Disaggregation & Heterogeneity

แต่ละบุคคลมีลักษณะต่างกัน → ผลกระทบอาจต่างกัน (heterogeneous treatment effects) ข้อมูลรวมมองไม่เห็นความแตกต่างนี้

Greater Information Content: ข้อมูลจุลภาคมีความแปรปรวน (variation) มากกว่าข้อมูลมหภาค → ประมาณค่าพารามิเตอร์ได้แม่นยำกว่า แต่ต้องระวังปัญหา heteroskedasticity, selection bias, measurement error

ปัญหาหลัก: Confounding & Unobserved Heterogeneity

Confounding Factors

ตัวแปรที่ส่งผลต่อทั้ง X และ Y พร้อมกัน ทำให้ correlation ไม่ใช่ causation

Cov(X, Y) ≠ 0 ⇏ X → Y

เช่น คนมีประกันสุขภาพใช้จ่ายด้านสุขภาพมากกว่า — แต่อาจเป็นเพราะคนป่วยบ่อยเลือกซื้อประกัน (selection)

Omitted Variable Bias (OVB)

ถ้าละเว้นตัวแปรที่สัมพันธ์กับทั้ง X และ Y:

E[β̂OLS] = β + γ · δ

โดย γ = ผลของตัวแปรที่ละเว้นต่อ Y, δ = ความสัมพันธ์ระหว่างตัวแปรที่ละเว้นกับ X

ทิศทาง bias ขึ้นอยู่กับเครื่องหมายของ γ × δ

ตัวอย่าง: ถ้า "ความร้ายแรงของโรค" ส่งผลให้ใช้จ่ายสุขภาพสูง (+) และทำให้ซื้อประกัน (+) → OVB จะทำให้ค่าสัมประสิทธิ์ของประกันถูก bias ขึ้น (overestimate)

Causal vs Noncausal Models

Noncausal (Predictive) Model

เป้าหมาย: ทำนาย Y ให้แม่นที่สุด

ไม่จำเป็นต้องตีความ β เป็น causal effect

E[Y | X] = Xβ

ใช้ได้ดีสำหรับ forecasting, risk scoring

เช่น ทำนายค่าใช้จ่ายสุขภาพจากอายุ เพศ จำนวนโรคเรื้อรัง — ไม่ต้องรู้ว่าตัวไหนเป็น "สาเหตุ"

Causal (Structural) Model

เป้าหมาย: วัดผลกระทบเชิงสาเหตุของ X ต่อ Y

ต้องใช้เครื่องมือพิเศษ:

  • Randomized experiment
  • Instrumental variables (Ch. 4)
  • Panel data / Fixed effects (Ch. 6)
  • Regression Discontinuity (Ch. 7)
  • Difference-in-Differences (Ch. 8)

Structural vs Reduced Form: Structural model สะท้อนทฤษฎีเศรษฐศาสตร์โดยตรง (เช่น demand & supply) → reduced form ได้จากการแก้ระบบสมการ ซึ่งแสดง endogenous variables เป็นฟังก์ชันของ exogenous variables เท่านั้น

Exogeneity & Conditional Independence

Weak Exogeneity

E[u | X] = 0

ค่าคาดหวังของ error term เท่ากับศูนย์เมื่อกำหนด X → เพียงพอสำหรับ OLS consistency

Strong Exogeneity

f(u | X) = f(u)

การกระจายตัวของ u ไม่ขึ้นกับ X เลย → เข้มงวดกว่า weak exogeneity

Conditional Independence Assumption (CIA)

เมื่อ control ตัวแปร W แล้ว:

Y(0), Y(1) ⊥ D | W

ผลลัพธ์ที่อาจเกิดขึ้น (potential outcomes) ไม่ขึ้นกับ treatment D เมื่อกำหนด W

→ Selection on observables: ถ้า CIA ถือ OLS + control W ก็เพียงพอสำหรับ causal inference

ตัวอย่าง: ถ้า suppins (ประกันเสริม) ถูกกำหนดโดย income, age, สุขภาพ — เมื่อเรา control ตัวแปรเหล่านี้แล้ว suppins จะ exogenous → OLS ให้ causal effect ได้

Linear Simultaneous Equations Model (SEM)

ตัวอย่าง: Supply & Demand

Qd = α0 + α1P + α2Income + ud
Qs = β0 + β1P + β2Cost + us
Qd = Qs (equilibrium)

ปัญหา Identification

ถ้า P เป็น endogenous (ถูกกำหนดพร้อมกับ Q) → OLS ให้ค่า bias

Order condition: ตัวแปร excluded instruments ≥ จำนวน endogenous regressors

Income อยู่ใน demand แต่ไม่อยู่ใน supply → identify supply ได้

Reduced Form

แก้ระบบเพื่อแสดง endogenous variables (P, Q) เป็นฟังก์ชันของ exogenous variables เท่านั้น:

P = π0 + π1Income + π2Cost + vP
Q = π3 + π4Income + π5Cost + vQ

Reduced form ประมาณค่าด้วย OLS ได้เสมอ (consistent)

สัญกรณ์และข้อตกลง (Notation)

ตัวแปรพื้นฐาน

สัญลักษณ์ความหมาย
yiตัวแปรตาม (dependent variable) ของหน่วยที่ i
xiเวกเตอร์ของตัวแปรอิสระ (1 × K)
βเวกเตอร์พารามิเตอร์ (K × 1)
uierror term (unobservable)
Nจำนวนหน่วยสังเกต (observations)

Linear Model

yi = xiβ + ui,   i = 1, ..., N

หรือในรูปเมทริกซ์:

y = Xβ + u

โดย y เป็น N×1, X เป็น N×K, β เป็น K×1, u เป็น N×1

สมมติฐาน OLS

  1. E[u | X] = 0 (exogeneity)
  2. rank(X) = K (no multicollinearity)
  3. Var(u | X) = σ²I (homoskedasticity)

3. OLS Regression

คุณสมบัติ OLS, Robust SE, การทดสอบสมมติฐาน

OLS: คุณสมบัติ & ตัวประมาณค่า

OLS Estimator

β̂OLS = (X'X)−1X'y

minimize ผลรวมของ residual ยกกำลังสอง:

minβ ∑i (yi − xiβ)2

คุณสมบัติ (ภายใต้ Gauss-Markov)

  • Unbiased: E[β̂] = β
  • Consistent: β̂ →p β เมื่อ N → ∞
  • BLUE: Best Linear Unbiased Estimator
  • Efficient: มี variance ต่ำสุดในกลุ่ม linear unbiased

Variance ของ OLS

Default (homoskedastic):

Var(β̂) = σ²(X'X)−1

Robust (Huber-White):

Var(β̂)HC = (X'X)−1 X'Ω̂X (X'X)−1

โดย Ω̂ = diag(ei²) ใช้ residuals ประมาณ heteroskedasticity

ใน Stata: เพิ่ม vce(robust) เพื่อใช้ Huber-White SE

ถ้ามี clustering: vce(cluster id)

Robust SE ให้ค่า consistent แม้มี heteroskedasticity → แนะนำให้ใช้เสมอ

Wald Test & การทดสอบสมมติฐาน

Wald Test

W = (Rβ̂ − r)' [R · Var(β̂) · R']−1 (Rβ̂ − r) ~ χ²(q)

โดย R = restriction matrix, r = restriction value, q = จำนวน restrictions

Stata แปลงเป็น F-test: F = W/q ~ F(q, N−K)

ตัวอย่างการทดสอบ

test phylim = actlim:

  • H0: βphylim = βactlim
  • H1: βphylim ≠ βactlim
  • ทดสอบว่า physical limitation กับ activity limitation มีผลต่อค่าใช้จ่ายเท่ากันหรือไม่

test phylim actlim totchr:

  • H0: βphylim = βactlim = βtotchr = 0
  • ทดสอบว่าตัวแปรสุขภาพทั้งหมดมีนัยสำคัญร่วมกันหรือไม่

DFITS คืออะไร?

DFITSi วัดผลกระทบของ observation i ต่อ fitted value ของตัวเอง

DFITSi = ei* √(hii / (1 − hii))

รวมทั้ง outlier (residual สูง) และ leverage (hii สูง)

กฎง่ายๆ (Rule of Thumb)

Observation ที่ |DFITS| > 2√((K+1)/N) ถือเป็น influential

ไม่ได้หมายความว่าต้องตัดทิ้ง แต่ควรตรวจสอบว่าเป็นข้อมูลจริงหรือ error

Box-Cox Transformation

* Box-Cox model: หา λ ที่เหมาะสมที่สุด boxcox totexp suppins phylim actlim totchr age female income if totexp>0, nolog

Box-Cox Transformation

y(λ) = (yλ − 1) / λ   (λ ≠ 0)
y(λ) = ln(y)   (λ = 0)

ทดสอบว่า λ ที่เหมาะที่สุดคืออะไร:

  • λ = 1 → ไม่ต้อง transform (levels)
  • λ = 0 → log transformation
  • λ = 0.5 → square root

ผลลัพธ์ Box-Cox

RESET Test (Ramsey)

* RESET test: ทดสอบ functional form quietly regress ltotexp suppins phylim actlim totchr age female income, vce(robust) estat ovtest

หลักการของ RESET

เพิ่มเทอม ŷ², ŷ³, ŷ&sup4; เข้าไปในสมการ:

y = Xβ + δ2ŷ² + δ3ŷ³ + δ4ŷ&sup4; + error

ถ้า reject → อาจมี nonlinearity ที่ไม่ได้รวมไว้

RESET Test Intuition

ถ้า linear fit ไม่พอ ŷ² จะมีนัยสำคัญ

  • ŷ อาจติดลบได้ในโมเดล levels → ปัญหาสำหรับตัวแปรที่ต้อง ≥ 0

Retransformation Problem: Log → Levels

ปัญหาหลัก

เมื่อประมาณค่า ln(y) = Xβ + u แล้วต้องการ E[y]:

exp(E[ln y | X]) ≠ E[y | X]

เพราะ Jensen's Inequality: สำหรับฟังก์ชัน convex (exp)

E[exp(z)] ≥ exp(E[z])

→ exp(ŷ) จะ underestimate E[y] เสมอ

Duanexp(Xβ̂) · (1/N)∑exp(ûi)u iid (ไม่ต้อง normal)

Jensen's Inequality: Intuition

ln y กระจายตัวแบบ symmetric (normal) แต่เมื่อ exp() กลับ ค่าสูงถูกขยายมากกว่าค่าต่ำ

→ ค่าเฉลี่ยของ exp(ln y) > exp(ค่าเฉลี่ยของ ln y)

6. Sampling Weights

Probability Weights & Weighted Regression

ทำไมต้องใช้ Sampling Weights?

Probability Weights (pweight)

ในการสำรวจ (survey) มักใช้ stratified sampling — บางกลุ่มถูกสุ่มมากกว่าสัดส่วนจริง (oversampled)

Sampling weight = จำนวนประชากรที่ observation นั้น "แทน"

Obspweightความหมาย
11,000แทนประชากร 1,000 คน
2500แทนประชากร 500 คน
350แทนประชากร 50 คน

เมื่อไรต้องใช้ Weight?

ถ้า sampling probability สัมพันธ์กับ Y (outcome) → ต้อง weight

เช่น oversampling ผู้ป่วยหนัก → ค่าเฉลี่ยค่ารักษาจะสูงเกินจริงถ้าไม่ weight

ถ้า sampling สุ่มล้วน (random) → weight ไม่จำเป็น

ในตัวอย่างนี้: สร้าง artificial pweight = totchr² + 0.5

→ คนที่มีโรคเรื้อรังมาก มี pweight สูง (แทนประชากรมากกว่า) เพราะถูก undersampled

→ survey oversampled คนป่วยน้อย (pweight ต่ำ) ค่าเฉลี่ยแบบ weighted จึงแตกต่างจาก unweighted

ผลกระทบของ Weights

  • Weighted mean: ให้น้ำหนักแต่ละ obs ตาม pweight → ค่าเฉลี่ยที่ represent ประชากรจริง
  • Weighted regression: ประมาณค่าพารามิเตอร์สำหรับประชากร (ไม่ใช่ sample)
  • Stata ใช้ pweight → robust SE อัตโนมัติ
