1. Introduction to Simulation

ทำไมต้อง "จำลองข้อมูล" ในเศรษฐมิติ?

Monte Carlo Experimentation

Monte Carlo Simulation คือเทคนิคการสร้างข้อมูลจำลองซ้ำ ๆ หลายรอบ เพื่อศึกษาคุณสมบัติของตัวประมาณค่า (estimator) ทางสถิติ

ตั้งชื่อตาม Casino Monte Carlo ในโมนาโก เนื่องจากใช้ ความน่าจะเป็น เป็นหัวใจสำคัญ

ทำไมต้อง Simulate?

  • ตรวจสอบว่า estimator มี unbiasedness หรือไม่
  • ดูว่า standard errors ที่คำนวณได้ถูกต้องไหม
  • ทดสอบว่า rejection rate ของ t-test ตรงตาม nominal size หรือไม่
  • เข้าใจพฤติกรรมของ estimator เมื่อ assumptions ถูกละเมิด

ขั้นตอนหลัก

  1. กำหนด DGP (Data Generating Process) — กำหนดสมการที่สร้างข้อมูล เช่น y = 1 + 2x + u
  2. สุ่มข้อมูล จาก DGP ตาม distribution ที่กำหนด
  3. ประมาณค่า ด้วยวิธีที่ต้องการ (เช่น OLS)
  4. เก็บผลลัพธ์ (coefficient, SE, t-stat)
  5. ทำซ้ำ 1,000–10,000 รอบ
  6. วิเคราะห์ distribution ของผลลัพธ์ทั้งหมด

Data Generating Process (DGP)

DGP คือ "กฎ" ที่กำหนดว่าข้อมูลถูกสร้างขึ้นมาอย่างไร — ในโลกจริงเราไม่รู้ DGP แต่ใน simulation เรา กำหนดเอง

ตัวอย่าง DGP แบบง่าย

\(y_i = \beta_0 + \beta_1 x_i + u_i\)

โดยที่:

  • \(\beta_0 = 1, \beta_1 = 2\) (ค่าพารามิเตอร์ที่แท้จริง)
  • \(x_i \sim \chi^2(1)\)
  • \(u_i \sim \chi^2(1) - 1\) (mean = 0 แต่ skewed)

ข้อได้เปรียบของ Simulation

เนื่องจากเรารู้ค่าแท้จริงของพารามิเตอร์ เราจึงสามารถ:

  • ตรวจสอบว่า \(\hat{\beta}_1\) มีค่าเฉลี่ยเท่ากับ 2 หรือไม่ (unbiasedness)
  • เปรียบเทียบ estimated SE กับ actual SD ของ \(\hat{\beta}_1\)
  • ดูว่าเมื่อ \(H_0: \beta_1 = 2\) เป็นจริง test ปฏิเสธ ~5% ไหม
* ตัวอย่าง DGP ใน Stata
clear all
set obs 100
generate x = rchi2(1)
generate u = rchi2(1) - 1
generate y = 1 + 2*x + u

ทำไม Simulation สำคัญในเศรษฐมิติ?

1. ทฤษฎี Asymptotic

ผลลัพธ์ทางทฤษฎีมักเป็น asymptotic — ใช้ได้เมื่อ n → ∞

Simulation ช่วยบอกว่า finite sample จะเป็นอย่างไร

2. ตรวจสอบ Estimators ใหม่

เมื่อพัฒนา estimator ใหม่ ก่อนใช้กับข้อมูลจริง ต้อง simulate เพื่อยืนยันว่าทำงานถูกต้อง

3. Power Analysis

คำนวณ statistical power ของ test ที่ซับซ้อน ซึ่งไม่มีสูตร closed-form

บทเรียนสำคัญ: Simulation ไม่ได้ "พิสูจน์" ทฤษฎี แต่เป็นเครื่องมือ ตรวจสอบ และ สร้างความเข้าใจ — ช่วยให้เห็นว่าทฤษฎีทำงานจริงอย่างไรในทางปฏิบัติ

2. Random Number Generation

Pseudorandom Numbers — หัวใจของ Simulation

Pseudorandom Number Generators (PRNG)

Pseudorandom numbers คือตัวเลขที่คอมพิวเตอร์สร้างจาก algorithm — ดู "สุ่ม" แต่จริง ๆ เป็น deterministic ถ้ารู้ seed

set seed — Reproducibility

การกำหนด seed ทำให้ได้ผลลัพธ์เหมือนกันทุกครั้งที่รัน

สำคัญมากสำหรับ replication ในงานวิจัย

set seed 10101 // กำหนด seed
set obs 10000
generate x = runiform() // U(0,1)
generate z = rnormal() // N(0,1)

คุณสมบัติที่คาดหวัง

  • Uniform: mean ≈ 0.5, SD ≈ 0.289
  • Normal: mean ≈ 0, SD ≈ 1
  • t(10): mean ≈ 0, heavier tails กว่า normal
  • Chi-sq(10): mean ≈ 10, skewed right
  • Poisson(5): mean ≈ 5, discrete

Histogram แบ่งข้อมูลเป็น bins แล้วนับจำนวนใน bin — วิธีง่ายที่สุดในการดู distribution

Negative Binomial จาก Gamma Mixing

Negative Binomial สามารถสร้างจาก Poisson-Gamma mixture:

  1. สุ่ม \(\delta_i \sim \text{Gamma}(1/\alpha, \alpha)\)
  2. ให้ \(\mu_i = \exp(x_i'\beta) \cdot \delta_i\)
  3. สุ่ม \(y_i \sim \text{Poisson}(\mu_i)\)

ผลลัพธ์คือ \(y_i \sim \text{NB}(\mu_i, \alpha)\)

ทำไม Negative Binomial?

Poisson สมมติ equidispersion: Var(y) = E(y)

ข้อมูลจริงมักมี overdispersion: Var(y) > E(y)

NB เพิ่ม heterogeneity ผ่าน \(\delta_i\) ทำให้ Var(y) = μ + αμ²

3. Central Limit Theorem

จาก Uniform สู่ Normal — พลังของการเฉลี่ย

Central Limit Theorem (CLT)

ทฤษฎีบท

ถ้า \(X_1, X_2, \ldots, X_n\) เป็น iid จาก distribution ใด ๆ ที่มี mean μ และ variance σ² แล้ว:

\(\sqrt{n}(\bar{X}_n - \mu) \xrightarrow{d} N(0, \sigma^2)\)

เมื่อ \(n \to \infty\)

ความหมาย: ไม่ว่าข้อมูลดั้งเดิมจะมี distribution อะไร ค่าเฉลี่ยของตัวอย่าง จะเข้าใกล้ Normal distribution เมื่อ sample size ใหญ่พอ

ตัวอย่าง: Uniform → Normal

  • สุ่ม n = 30 ตัวจาก Uniform(0,1)
  • คำนวณค่าเฉลี่ย \(\bar{X}\)
  • ทำซ้ำ 10,000 ครั้ง
  • Distribution ของ \(\bar{X}\) จะเป็น Normal!
\(\bar{X} \approx N\left(0.5, \frac{1/12}{30}\right)\)
program onesample, rclass
  drop _all
  set obs 30
  generate x = runiform()
  summarize x
  return scalar meanforonesample = r(mean)
end

Stata: คำสั่ง simulate

simulate เป็นคำสั่งที่ช่วยทำ Monte Carlo ได้ง่าย — เรียก program ซ้ำ ๆ แล้วเก็บผลลัพธ์อัตโนมัติ

* เรียก program onesample ซ้ำ 10,000 ครั้ง
simulate xbar = r(meanforonesample), seed(10101) reps(10000): onesample

* ดูผลลัพธ์
summarize xbar
histogram xbar, normal title("Distribution of Sample Mean")

ผลลัพธ์ที่คาดหวัง

  • Mean ของ xbar ≈ 0.5 (= μ ของ Uniform)
  • SD ของ xbar ≈ 0.0527 (= σ/√n = √(1/12)/√30)
  • Histogram มีรูปร่าง bell-shaped

โครงสร้าง simulate

  • reps: จำนวนรอบ
  • program: program ที่ต้อง return scalar

4. Simulation for Regression

OLS กับ errors ที่ไม่เป็น Normal — ยังใช้ได้ไหม?

DGP: OLS กับ Chi-squared Errors

Data Generating Process

\(y_i = 1 + 2x_i + u_i\)

โดยที่: \(x_i \sim \chi^2(1)\),  \(u_i = \chi^2(1) - 1\) (mean=0, skewed)

True values: \(\beta_0 = 1, \beta_1 = 2\)

ทำไมใช้ Chi-squared Error?

ทดสอบว่า OLS ยังทำงานดีเมื่อ error ไม่เป็น Normal — CLT บอกว่า \(\hat{\beta}\) ยังควรจะ approximately normal

program chi2data, rclass
  drop _all
  set obs $numobs
  generate x = rchi2(1)
  generate u = rchi2(1) - 1
  generate y = 1 + 2*x + u
  regress y x
  return scalar b2 = _b[x]
  return scalar se2 = _se[x]
  return scalar t2 = (_b[x]-2)/_se[x]
  return scalar r2 = abs(return(t2)) > invttail($numobs-2,.025)
end

ตัวแปรที่เก็บ

b2: \(\hat{\beta}_1\)  |  se2: SE ของ \(\hat{\beta}_1\)  |  t2: t-stat (H0: β1=2)  |  r2: reject indicator

เรียก simulate — ทำ 1,000 รอบ

global numobs = 100
simulate b2f=r(b2) se2f=r(se2) t2f=r(t2) reject2f=r(r2), reps(1000): chi2data
summarize b2f // ตรวจ unbiasedness: mean ควร ≈ 2
summarize se2f // ตรวจ SE accuracy: mean ≈ SD of b2f
summarize reject2f // ตรวจ test size: mean ควร ≈ 0.05
histogram t2f, normal // เปรียบเทียบ t-stat กับ theoretical dist

Unbiasedness

ถ้า mean ของ \(\hat{\beta}_1\) ≈ 2 → OLS เป็น unbiased

SE Accuracy

ถ้า mean(SE) ≈ SD(\(\hat{\beta}_1\)) → SE formula ถูกต้อง

Test Size

ถ้า rejection rate ≈ 5% → t-test มี correct size

การเปรียบเทียบ t-statistic กับ Theoretical Distribution

ถ้า OLS assumptions เป็นจริงทั้งหมด (รวมถึง normality ของ error):

\(t = \frac{\hat{\beta}_1 - \beta_1}{SE(\hat{\beta}_1)} \sim t(n-2)\)

แต่เมื่อ error เป็น (\chi^2(1) - 1) (ไม่ใช่ normal) CLT ทำให้:

\(t \xrightarrow{d} N(0,1) \text{ เมื่อ } n \text{ ใหญ่พอ}\)

สิ่งที่ต้องตรวจสอบ

  • Histogram ของ t-statistic ควรมีรูปร่างคล้าย bell curve
  • n เล็ก (<30): t-distribution อาจไม่ approximate ได้ดี เพราะ error เบ้มาก
  • n ใหญ่ (≥100): CLT ทำงานได้ดี แม้ error จะ skewed

ผลลัพธ์ที่ดี

  • Mean(b2) ≈ 2.00
  • Mean(SE) ≈ SD(b2)
  • Rejection rate ≈ 0.05
  • t-stat histogram ซ้อนกับ N(0,1) ได้ดี

5. Endogenous Regressors

เมื่อ x สัมพันธ์กับ u — OLS ล้มเหลว

ปัญหา Endogeneity

DGP ที่มี Endogeneity

\(y_i = 1 + 2x_i + u_i\)

โดยที่ \(\text{Corr}(x_i, u_i) = \rho \neq 0\)

สร้างได้โดย:

  • \(x_i = \pi z_i + v_i\)
  • \(u_i = \gamma v_i + e_i\)
  • \(z_i\) เป็น instrument (ไม่สัมพันธ์กับ u)

ผลของ Endogeneity

เมื่อ \(\text{Cov}(x, u) \neq 0\):

  • OLS เป็น biased — \(E(\hat{\beta}_{OLS}) \neq \beta\)
  • OLS เป็น inconsistent — bias ไม่หายไปแม้ n → ∞
  • \(\text{plim } \hat{\beta}_{OLS} = \beta + \frac{\text{Cov}(x,u)}{\text{Var}(x)}\)

ทางแก้: IV Estimation

ใช้ Instrumental Variable (IV) ที่สัมพันธ์กับ x แต่ไม่สัมพันธ์กับ u

\(\hat{\beta}_{IV}\) เป็น consistent

Simulation: OLS vs IV เมื่อมี Endogeneity

* DGP: y = 1 + 2*x + u, Corr(x,u) ≠ 0
program endogdata, rclass
  drop _all
  set obs 200
  generate z = rnormal() // instrument
  generate v = rnormal() // ส่วนที่ทำให้ endog
  generate e = rnormal()
  generate x = 1*z + v // x ขึ้นกับ z และ v
  generate u = 0.8*v + e // u ขึ้นกับ v → Corr(x,u)≠0
  generate y = 1 + 2*x + u
  regress y x
  return scalar b_ols = _b[x]
  ivregress 2sls y (x = z)
  return scalar b_iv = _b[x]
end

ผลที่คาดหวัง: Mean(β̂OLS) ≠ 2 (biased) แต่ Mean(β̂IV) ≈ 2 (consistent)

ผลของ Endogeneity: OLS Inconsistency

ข้อสังเกตจาก Simulation

  • OLS: histogram ของ \(\hat{\beta}_1\) จะ ไม่ centered ที่ 2
  • IV: histogram ของ \(\hat{\beta}_1\) จะ centered ที่ 2 แต่กว้างกว่า
  • เมื่อเพิ่ม n: OLS bias ไม่ลดลง (inconsistent)
  • เมื่อเพิ่ม n: IV variance ลดลง (consistent)

Bias-Variance Trade-off

IV มี variance สูงกว่า OLS เสมอ (ราคาของ consistency)

แต่ในกรณี endogeneity:

  • OLS: low variance แต่ biased (ไม่ดี)
  • IV: higher variance แต่ unbiased/consistent (ดีกว่า)

ยิ่ง ρ สูง → OLS bias ยิ่งมาก → IV ยิ่งจำเป็น

Key Takeaways

สิ่งที่ได้เรียนรู้

  1. Monte Carlo ช่วยตรวจสอบคุณสมบัติของ estimator ที่ทฤษฎีพยากรณ์ไว้
  2. CLT ทำให้ OLS ใช้ได้แม้ error ไม่เป็น Normal
  3. set seed เป็นสิ่งจำเป็นสำหรับ reproducibility
  4. Endogeneity ทำให้ OLS inconsistent — ต้องใช้ IV
