1. Introduction to GLS

เมื่อ OLS ไม่ใช่ตัวประมาณค่าที่ดีที่สุดอีกต่อไป

ทบทวน: สมมติฐานของ OLS

Gauss-Markov Assumptions

OLS เป็น BLUE (Best Linear Unbiased Estimator) เมื่อ:

  • E[u|X] = 0 (exogeneity)
  • Var[u|X] = \sigma^2 I_n (spherical errors)

สมมติฐานข้อสอง = homoskedasticity + noncorrelation

ถ้าสมมติฐานข้อสองผิด?

OLS ยังคง consistent แต่ไม่ efficient อีกต่อไป และ default SE จะ ผิดพลาด

กรณีที่ spherical errors ล้มเหลว:

  • Heteroskedasticity: Var[u_i|x_i] = \sigma_i^2 แปรผันตาม x
  • Correlation: Cov[u_i, u_j|X] \neq 0
  • ทั้งสอง: Var[u|X] = \Omega \neq \sigma^2 I

ตัวอย่างในทางปฏิบัติ:

  • ค่าใช้จ่ายทางการแพทย์ — ผู้มีรายได้สูงมี variance สูงกว่า
  • ข้อมูล cross-equation — ค่าความผิดพลาดของสมการต่าง ๆ สัมพันธ์กัน
  • ข้อมูลสำรวจ — clustering ทำให้ errors สัมพันธ์กัน

GLS: แนวคิดหลัก

Generalized Least Squares (GLS) แก้ปัญหาโดยการ transform model ให้กลับมามี spherical errors

โมเดลเดิม:

y = X\beta + u, \quad Var[u|X] = \Omega

Transform ด้วย \Omega^{-1/2}:

\Omega^{-1/2}y = \Omega^{-1/2}X\beta + \Omega^{-1/2}u

โมเดลใหม่: y^* = X^*\beta + u^*

Var[u^*|X] = \Omega^{-1/2}\Omega\Omega^{-1/2} = I

GLS estimator:

\hat{\beta}_{GLS} = (X'\Omega^{-1}X)^{-1}X'\Omega^{-1}y

คือ OLS ที่ทำบน transformed model

ถ้า \Omega ถูกกำหนดถูกต้อง GLS จะ efficient กว่า OLS

ในทางปฏิบัติ: เราไม่รู้ \Omega จริง ต้องประมาณค่า

จึงใช้ FGLS (Feasible GLS) — ใช้ \hat{\Omega} แทน \Omega

บทนี้ครอบคลุม 3 หัวข้อหลัก

Heteroskedasticity

ความแปรปรวนไม่คงที่

Var[u_i|x_i] = \sigma^2(x_i)

แก้ด้วย WLS/FGLS

SUR

สมการร่วม (Seemingly Unrelated Regressions)

Cov[u_{1i}, u_{2i}] \neq 0

แก้ด้วย sureg

Survey Data

ข้อมูลสำรวจ

clustering + stratification + weighting

แก้ด้วย svyset + svy:

แนวคิดร่วม: ทั้ง 3 กรณีมีปัญหาคือ Var[u|X] \neq \sigma^2 I — OLS ไม่ efficient และ default SE ผิดพลาด

2. Heteroskedasticity & FGLS

เมื่อ variance ขึ้นกับตัวแปรอิสระ

Heteroskedasticity คืออะไร?

Homoskedasticity: Var[u_i|x_i] = \sigma^2 (คงที่)

Heteroskedasticity: Var[u_i|x_i] = \sigma^2(x_i) (แปรผัน)

Skedasticity function: \sigma^2(x_i) คือฟังก์ชันที่อธิบายว่า variance เปลี่ยนตาม x อย่างไร

ผลกระทบของ heteroskedasticity:

  • OLS ยังคง unbiased และ consistent
  • แต่ไม่ efficient (ไม่ใช่ BLUE)
  • Default SE ผิด → t-test, F-test ผิดพลาด

ตัวอย่าง: Generated Data

DGP (Data Generating Process):

y = 1 + x_2 + x_3 + u, \quad u = \sqrt{\exp(-1 + 0.2x_2)} \cdot e

โดย x_2, x_3, e \sim N(0, 5^2) — ดังนั้น Var[u|x] = \exp(-1 + 0.2x_2) (heteroskedastic ใน x_2)

* สร้างข้อมูล heteroskedastic
set seed 10101
quietly set obs 500
generate double x2 = 5*rnormal(0)
generate double x3 = 5*rnormal(0)
generate double e = 5*rnormal(0)
generate double u = sqrt(exp(-1+0.2*x2))*e
generate double y = 1 + 1*x2 + 1*x3 + u

สอดคล้องกับ DGP: Var[u|x] = \exp(-1 + 0.2x_2)

FGLS: ขั้นตอนที่ 1 — ประมาณ Skedasticity Function

Feasible GLS (FGLS) มี 2 ขั้นตอน: (1) ประมาณค่า \sigma^2(x_i) แล้ว (2) ใช้ WLS ด้วยน้ำหนัก w_i = 1/\hat{\sigma}^2(x_i)

Step 1: ประมาณ skedasticity function

* OLS เพื่อได้ residuals
quietly regress y x2 x3
predict double uhat, resid
generate double uhatsq = uhat^2

* NLS: uhatsq = exp(a0 + a1*x2)
generate double one = 1
nl (uhatsq = exp({xb: x2 one})), nolog
predict double varu, yhat

ทำไมใช้ exp()?

เพราะ variance ต้องเป็น บวกเสมอ

Exponential function รับประกันว่า \hat{\sigma}^2(x) = \exp(\hat{\alpha}_0 + \hat{\alpha}_1 x_2) > 0

ผลลัพธ์จาก NLS:

  • \hat{\alpha}_0 \approx 2.22 (ค่าจริง = -1 + ln(25) = 2.22)
  • \hat{\alpha}_1 \approx 0.20 (ค่าจริง = 0.20)

NLS สามารถกู้คืนค่าจริงได้!

3. Seemingly Unrelated Regressions

เมื่อความผิดพลาดของหลายสมการสัมพันธ์กัน

SUR: แนวคิดพื้นฐาน

Seemingly Unrelated Regressions (SUR)

ระบบ m สมการสำหรับ individual i:

y_{mi} = x_{mi}'\beta_m + u_{mi}

แต่ละสมการอาจมี regressors ต่างกัน

จุดสำคัญ: errors ข้ามสมการสัมพันธ์กัน

E[u_{mi} \cdot u_{ni}] = \sigma_{mn} \neq 0 \text{ for } m \neq n

"Seemingly unrelated" เพราะดูเหมือนไม่เกี่ยวกัน แต่จริง ๆ มี correlation ผ่าน errors

ทำไม SUR ดีกว่า OLS แยกสมการ?

  • ถ้า \sigma_{mn} \neq 0 → SUR (GLS) efficient กว่า OLS
  • SUR ใช้ข้อมูลจาก ความสัมพันธ์ข้ามสมการ
  • ยิ่ง correlation สูง → SUR ยิ่งดีกว่า OLS

กรณีพิเศษ: ถ้า regressors เหมือนกันทุกสมการ

SUR = OLS แยกสมการ (ไม่มี efficiency gain)

  (ltotothr age age2 educyr actlim totchr private), corr

จึงมี efficiency gain จาก SUR

Regressors ต่างกันบางส่วน:

  • สมการ 1 มี medicaid (สมการ 2 ไม่มี)
  • สมการ 2 มี educyr (สมการ 1 ไม่มี)

Bootstrap ให้ SE ที่ robust ต่อ heteroskedasticity

เทียบเท่ากับ WLS ในกรณี single equation

Cross-equation Tests

* ทดสอบ age, age2 ในทั้ง 2 สมการ
test age age2

* ทดสอบ age, age2 เฉพาะสมการ 1
test [ldrugexp]age [ldrugexp]age2

* ทดสอบข้ามสมการ
test [ldrugexp]private = [ltotothr]private

Cross-equation test เป็นจุดแข็งของ SUR

สามารถทดสอบว่า effect ของ private insurance เท่ากันหรือไม่ในทั้ง 2 สมการ

SUR: Constraints ข้ามสมการ

* กำหนด constraint ข้ามสมการ
constraint 1 [ldrugexp]private = [ltotothr]private

* ประมาณค่าภายใต้ constraint
sureg (ldrugexp age age2 actlim totchr medicaid private) ///
  (ltotothr age age2 educyr actlim totchr private), constraints(1)

ประโยชน์ของ Constrained SUR:

  • ถ้า constraint ถูก → efficient กว่า unconstrained
  • บังคับให้ผลของ private insurance เท่ากันทั้ง 2 สมการ
  • ใช้ข้อมูลร่วมกันอย่างมีประสิทธิภาพ

หลักการ: SUR เป็น GLS ในระดับ system of equations

\Omega คือ block-diagonal matrix ของ \Sigma \otimes I_n

โดย \Sigma เป็น m \times m cross-equation covariance matrix

4. Survey Data

Weighting, Clustering, และ Stratification

ทำไมข้อมูลสำรวจต้องวิเคราะห์แบบพิเศษ?

Complex Survey Design มี 3 องค์ประกอบ:

  • Weighting (pweight): แต่ละคนมีโอกาสถูกเลือกไม่เท่ากัน
  • Clustering (PSU): สุ่มเป็นกลุ่ม ไม่ใช่สุ่มรายคน
  • Stratification: แบ่งประชากรเป็นชั้น สุ่มภายในชั้น

ถ้าไม่ปรับ:

  • Point estimates อาจ biased (ไม่มี weight)
  • SE อาจ ต่ำเกินไป (ไม่ปรับ clustering)
  • SE อาจ สูงเกินไป (ไม่ปรับ stratification)

ตัวอย่าง: NHANES II

National Health and Nutrition Examination Survey

  • สุ่มแบบ multi-stage cluster sampling
  • PSU = primary sampling unit (เช่น county)
  • Strata = geographic regions
  • pweight = probability weight (inverse of selection probability)
use nhanes2.dta
svyset psu [pweight=finalwgt], strata(strata)

Survey Design Effects: Interactive

Design Effect = Var(survey) / Var(SRS)

0.10
20

Design Effect (DEFF):

DEFF \approx 1 + (m-1)\rho

โดย m = cluster size, \rho = intraclass correlation

DEFF = 2.90

SE(survey) = 1.70 x SE(SRS)

Effective sample size = 34% ของ nominal

Clustering: ทำไม SE ถึงเพิ่มขึ้น

Clustering = การสุ่มเป็นกลุ่ม

คนในกลุ่มเดียวกัน (PSU) มีลักษณะคล้ายกัน

→ observations ไม่ independent

→ "effective sample size" น้อยกว่า nominal

ผลลัพธ์:

SE จาก survey > SE จาก SRS

ถ้า ICC = 0.1 และ cluster size = 20

DEFF = 1 + 19(0.1) = 2.9

SE เพิ่มขึ้น \sqrt{2.9} \approx 1.7 เท่า

Stratification: SE ลดลง

Stratification = แบ่งประชากรเป็นชั้นก่อนสุ่ม

ทำให้ sample "representative" มากขึ้น

→ ลด sampling variability

→ SE ลดลง

ผลสุทธิ:

Clustering เพิ่ม SE ↑

Stratification ลด SE ↓

ผลสุทธิขึ้นกับ design — ใน NHANES II clustering มีผลมากกว่า จึง SE สุทธิเพิ่มขึ้น

คำถาม: Clustering ในข้อมูลสำรวจมีผลอย่างไรต่อ SE?

Quiz: GLS ภาพรวม

คำถาม: ข้อใดเป็นจริงเกี่ยวกับ WLS (Weighted Least Squares)?

ก. WLS ต้องใช้ skedasticity function ที่ถูกต้องจึงจะ consistent
ข. WLS ให้ SE ที่ถูกต้องเสมอแม้ไม่ใช้ robust SE
ค. WLS = FGLS + robust SE ทำให้ได้ทั้ง efficiency gain และ inference ที่ถูกต้อง
ง. WLS ใช้ได้เฉพาะกับ heteroskedasticity ไม่ใช้กับ SUR
