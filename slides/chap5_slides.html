1. Introduction

ทำไมเราต้องมองมากกว่า "ค่าเฉลี่ย"?

OLS เล่าเรื่องได้แค่ครึ่งเดียว

OLS (Ordinary Least Squares) ประมาณค่า conditional mean — ค่าเฉลี่ยของ y เมื่อกำหนด x

แต่ "ค่าเฉลี่ย" เป็นเพียง มุมมองเดียว ของ distribution ทั้งหมด

Quantile Regression (QR) ให้เราศึกษาความสัมพันธ์ระหว่าง y กับ x ที่ ทุกจุด ของ conditional distribution

เช่น ผลกระทบต่อคนที่มี ค่าใช้จ่ายสูง (quantile 90) อาจต่างจากคนที่มี ค่าใช้จ่ายต่ำ (quantile 10)

ลองดู: OLS vs QR

ข้อดีของ Quantile Regression

Robust ต่อ Outliers

Median regression (LAD) ทนต่อค่าผิดปกติได้ดีกว่า OLS เพราะ minimize absolute deviations แทน squared

เห็น Location + Scale

ศึกษาผลกระทบต่อทั้ง "ตำแหน่ง" และ "การกระจาย" ของ distribution ไปพร้อมกัน

Semiparametric

ไม่ต้อง assume parametric distribution ของ error terms → เหมาะกับ heteroskedastic data

ไม่ต้องการ E(y|x): บาง distribution มีหางหนามาก (เช่น Cauchy) ถ้าเราสุ่มตัวอย่างเพิ่มไปเรื่อยๆ sample mean จะไม่ converge — เพราะยิ่งสุ่มมากยิ่งเจอค่าสุดขีดที่ดึง mean ให้เปลี่ยนตลอด (population mean ไม่มีจริง) → OLS ที่ประมาณ mean จึงใช้ไม่ได้ แต่ median ยังหาได้เสมอ เพราะแค่นับตำแหน่งกลาง ไม่สนว่าค่าสุดขีดจะใหญ่แค่ไหน → QR จึง consistent ได้เสมอ

Equivariance: ถ้าประมาณ QR ของ ln(y) แล้ว exp() กลับ จะได้ quantile ของ y โดยตรง เพราะ Qq(h(y)) = h(Qq(y)) สำหรับ h() monotone — OLS ทำไม่ได้เพราะ E(h(y)) ≠ h(E(y))

ลองดู: Mean vs Median เมื่อหางหนา

Cauchy distribution (หางหนา — mean ไม่ exist)

Normal distribution (หางบาง — mean exist)

n = 0

สังเกต Cauchy: เส้น Mean (แดง) กระโดดไปมาไม่หยุด แม้ n จะมากขึ้น เพราะ extreme values ดึงตลอด — แต่เส้น Median (น้ำเงิน) นิ่งเข้าหา 0

สังเกต Normal: ทั้ง Mean และ Median converge เข้าหา 0 อย่างสงบ เพราะหางบางไม่มีค่าสุดขีด — OLS ใช้ได้ดีในกรณีนี้

2. Theoretical Background

Objective Function ของ Quantile Regression

Quantile คืออะไร?

Quantile q คือจุดที่แบ่ง distribution โดย q×100% ของข้อมูลอยู่ต่ำกว่าจุดนั้น

  • q = 0.25 → Q1 (25th percentile)
  • q = 0.50 → Median
  • q = 0.75 → Q3 (75th percentile)

ตัวอย่าง: ถ้า quantile 0.90 ของค่ารักษาพยาบาล = 50,000 บาท หมายความว่า 90% ของคนมีค่ารักษาต่ำกว่า 50,000 บาท

ลองเลื่อน quantile ดู

0.50

Objective Function ของ QR

QR estimator สำหรับ quantile q จะ minimize:

Q(βq) = ∑i: yi ≥ xiβq q |yi − xiβq| + ∑i: yi < xiβq (1−q) |yi − xiβq|

โดยที่ 0 < q < 1 ซึ่งเราใช้ βq แทน β เพื่อเน้นว่าค่า q ต่างกันจะได้ β ต่างกัน

Check function ρq(u):

ρq(u) = u·(q − 1[u < 0])

  • ถ้า u ≥ 0: weight = q
  • ถ้า u < 0: weight = (1−q)

เมื่อ q = 0.5 → weight เท่ากันทั้งสองฝั่ง = median regression

Check Function ρq(u)

0.50

Objective Function: OLS vs QR

OLS minimize:

∑i (yi − xiβ)2

Squared loss → symmetric ทุก residual ถูก weight เท่ากัน

Outlier มี penalty สูงมาก (error ยกกำลัง 2)

QR minimize:

∑i ρq(yi − xiβq)

Absolute loss + asymmetric weight:

  • จุด เหนือ เส้น (yi > xiβq): penalty = q × |residual|
  • จุด ใต้ เส้น (yi < xiβq): penalty = (1−q) × |residual|

ตัวอย่าง q = 0.75:

  • จุดเหนือเส้น → penalty × 0.75 (สูง)
  • จุดใต้เส้น → penalty × 0.25 (ต่ำ)

→ เส้นถูก "ดึง" ขึ้นจนมี ~75% ของข้อมูลอยู่ต่ำกว่า เพราะ cost ฝั่งบนสูงกว่า 3 เท่า

Intuition สำคัญ: ค่า q กำหนดว่าเส้นจะ "ตัด" distribution ตรงไหน โดยทำให้ข้อมูลฝั่งที่มี penalty สูงกว่าถูก minimize ก่อน

กรณีพิเศษของค่า q ต่างๆ

q = 0.50 (Median / LAD):

weight เท่ากันทั้งสองฝั่ง (0.5 : 0.5) → เส้นผ่ากลาง distribution = Least Absolute Deviations

Robust กว่า OLS เพราะใช้ absolute (ไม่ใช่ squared)

q = 0.10 (Lower tail):

เหนือเส้น: penalty = 0.10 (ต่ำ) ← ไม่แคร์จุดเหนือมาก

ใต้เส้น: penalty = 0.90 (สูง) ← แคร์จุดใต้มาก

→ เส้นถูกดึง ลงต่ำ เพื่อให้มีจุดใต้เส้นน้อยที่สุด (~10%)

q = 0.90 (Upper tail):

เหนือเส้น: penalty = 0.90 (สูง) ← แคร์จุดเหนือมาก

ใต้เส้น: penalty = 0.10 (ต่ำ) ← ไม่แคร์จุดใต้มาก

→ เส้นถูกดึง ขึ้นสูง เพื่อให้มีจุดเหนือเส้นน้อยที่สุด (~10%)

สรุปหลักการ: ที่ quantile q ใดๆ QR จะ fit เส้นที่ทำให้สัดส่วนข้อมูลใต้เส้น ≈ q — ดังนั้นการเปลี่ยน q จะเลื่อนเส้นไปตำแหน่งต่างๆ ของ conditional distribution

0.50

สรุป: QR ใช้ absolute deviation (ไม่ใช่ squared) จึง robust ต่อ outlier และใช้ asymmetric weight จึง หาตำแหน่ง quantile ที่ต้องการได้ — ต่างจาก OLS ที่ใช้ squared + symmetric จึงได้แค่ mean

Equivariance: ถ้าประมาณ QR ของ ln(y) แล้ว transform กลับด้วย exp() จะได้ quantile ของ y โดยตรง — OLS ทำแบบนี้ไม่ได้!

Equivariance: เปรียบเทียบ QR vs OLS ทีละขั้น

สมมติเรามีข้อมูลค่ารักษาพยาบาล (y) ซึ่ง right-skewed มาก จึง log ก่อนวิเคราะห์

ขั้นตอนQR (Median)OLS (Mean)
1. ประมาณใน log scale Median ของ ln(y)|x = 8.50 Mean ของ ln(y)|x = 8.30
2. Transform กลับด้วย exp() exp(8.50) = 4,915 exp(8.30) = 4,023
3. ค่าที่ได้คืออะไร? = Median ของ y|x โดยตรง! ≠ Mean ของ y|x (biased!)
4. ทำไม? Q.5(exp(ln y)) = exp(Q.5(ln y))
quantile commutes กับ monotone transform		 E(exp(ln y)) ≠ exp(E(ln y))
mean ไม่ commute (Jensen's inequality)
5. ถ้าต้องการ Mean ของ y|x จริงๆ? ไม่ใช่เป้าหมายของ QR ต้องแก้ bias ด้วย Duan's smearing หรือ retransformation อื่น

พิสูจน์: Qq(h(Y)) = h(Qq(Y))

กำหนดให้: h() เป็น strictly increasing function (เช่น exp, log, sqrt)

นิยาม: Qq(Y) = c ที่ทำให้ P(Y ≤ c) = q

พิสูจน์:

ให้ c = Qq(Y)  →  P(Y ≤ c) = q     ... (1)

เพราะ h() strictly increasing:   Y ≤ c ⇔ h(Y) ≤ h(c)     ... (key step)

ดังนั้น P(h(Y) ≤ h(c)) = P(Y ≤ c) = q     จาก (1)

นั่นคือ h(c) เป็น quantile ที่ q ของ h(Y)  →  Qq(h(Y)) = h(c) = h(Qq(Y))   ▢

Equivariance: ตัวอย่าง + ทำไม Mean ทำไม่ได้

ตัวอย่าง: h = exp

ให้ Y = ln(ค่ารักษา) จากพิสูจน์:

Q.5(exp(ln(ค่ารักษา))) = exp(Q.5(ln(ค่ารักษา)))

→ Median ค่ารักษา = exp(Median ของ log ค่ารักษา)

ใช้ได้กับทุก quantile q ไม่ใช่แค่ median

ทำไม Mean ทำไม่ได้?

ในพิสูจน์ เราใช้ว่า Y ≤ c ⇔ h(Y) ≤ h(c)

Mean ไม่ได้ถูกนิยามจาก "ตำแหน่ง" แบบนี้ — Mean ขึ้นกับ ทุกค่า ในข้อมูลรวมกัน

Jensen's inequality: สำหรับ convex h (เช่น exp):

E(h(Y)) ≥ h(E(Y))

ดังนั้น exp(E(ln y)) ≤ E(y) เสมอ → OLS ที่ exp() กลับจะ underestimate mean

Equivariance: สรุป Intuition

หลักการ

สำหรับ monotone function h() (เช่น exp, log, sqrt):

  • Quantile: Qq(h(y)) = h(Qq(y)) ← เสมอจริง
  • Mean: E(h(y)) ≠ h(E(y)) ← ไม่จริงโดยทั่วไป

เพราะ quantile คือ "ตำแหน่ง" ในลำดับ — ถ้า transform ไม่เปลี่ยนลำดับ ตำแหน่งก็ไม่เปลี่ยน

นัยสำคัญในทางปฏิบัติ

  • ค่ารักษาพยาบาล → ประมาณ QR ของ ln(y) แล้ว exp() กลับได้ทันที
  • รายได้ → เหมือนกัน ไม่ต้องแก้ bias
  • ราคาบ้าน → เหมือนกัน

ทุกครั้งที่ใช้ log-transformed dependent variable → QR ตีความง่ายกว่า OLS เพราะ transform กลับได้โดยไม่มี bias

ทดสอบ Heteroskedasticity & Coefficient Equality

* Breusch-Pagan test
quietly regress ltotexp suppins totchr age female white
estat hettest suppins totchr age female white, iid

ผลลัพธ์: ถ้า p-value < 0.05 → reject H0 ของ homoskedasticity → QR จะให้ข้อมูลเพิ่มเติมที่ OLS ไม่สามารถเห็นได้

* Simultaneous QR + test
set seed 10101

* Test: coefficients equal across quantiles?
test [q25=q50=q75]: suppins

sqreg ประมาณ QR หลาย quantile พร้อมกัน ได้ joint variance-covariance matrix ที่ถูกต้อง สำหรับทดสอบ coefficient equality ข้าม quantile

H0: β0.25 = β0.50 = β0.75

  • เส้นทึบ = QR coefficient
  • เส้นประ = OLS coefficient (คงที่)
  • Band = Bootstrap confidence interval

ถ้าเส้น QR เป็นเส้นตรงราบ = ผลกระทบเท่ากันทุก quantile (homogeneous effect)

ถ้าเส้น QR เอียงหรือโค้ง = ผลกระทบต่างกันตาม quantile (heterogeneous effect)

Checklist: วิธีอ่านผล QR (1/2)

Step 1: ตรวจว่าควรใช้ QR หรือไม่

  • รัน OLS แล้วทดสอบ estat hettest
  • Reject → มี heteroskedasticity → QR มีประโยชน์
  • ดู quantile plot (qplot) ว่า distribution เบ้หรือมี outliers หรือไม่

Step 2: เปรียบเทียบ coefficient ข้าม quantile

  • รัน sqreg ที่ q = .25, .50, .75 พร้อมกัน
  • ทดสอบ test [q25=q50=q75]: varname
  • Reject → coefficient ต่างกัน → heterogeneous effect
  • Fail to reject → effect ค่อนข้างเท่ากัน → OLS อาจเพียงพอ

Checklist: วิธีอ่านผล QR (2/2)

Step 3: ตีความกราฟ grqreg

  • เส้น QR ราบ = effect เท่ากันทุก quantile
  • เส้น QR เพิ่มขึ้น = effect แรงขึ้นที่ upper tail
  • เส้น QR ลดลง = effect แรงที่ lower tail
  • CI กว้างขึ้น ที่ quantile สูง/ต่ำ = precision ต่ำ

Step 4: ตีความเชิง policy

  • Effect ต่างกันข้าม quantile → นโยบาย "one-size-fits-all" ไม่เหมาะ
  • ใช้ bootstrap SE (bsqreg) เสมอ ไม่ใช่ asymptotic SE จาก qreg
  • รายงานผลหลาย quantile (เช่น .25, .50, .75) เพื่อให้เห็นภาพรวม

4. Simulation: Heteroskedastic Data

เข้าใจ QR ผ่านข้อมูลจำลอง

Data Generating Process (DGP)

สร้างข้อมูลที่ conditional mean ขึ้นกับ x2 และ x3 แต่ conditional variance ขึ้นกับ x2 เท่านั้น:

y = 1 + 1·x2 + 1·x3 + u
u = (0.1 + 0.5·x2) · e     โดย e ~ N(0, 25)

Conditional quantile ที่ q:

Qq(y|x) = 1 + (1 + 0.5·F-1e(q))·x2 + 1·x3

สังเกต:

  • Coef ของ x3 = 1 เสมอ (ไม่อยู่ใน hetero function จึงไม่เปลี่ยนตาม q)
  • Coef ของ x2 = 1 + 0.5·F-1e(q) เปลี่ยนตาม q เพราะ x2 อยู่ใน u
set seed 10101
set obs 10000
generate x2 = rchi2(1)
generate x3 = 5*rnormal(0)
generate e = 5*rnormal(0)
generate u = (.1+0.5*x2)*e
generate y = 1 + 1*x2 + 1*x3 + u

ทำไม x2 ~ Chi2(1)? เพราะ hetero function (0.1 + 0.5x2) ต้องการ x2 ≥ 0 เสมอ เพื่อให้ variance เป็นบวก Chi2 distribution รับประกันเรื่องนี้

* Test: x3 coefficient equal at q=.25 and q=.75?
test [q25]x3 = [q75]x3   // expect: FAIL TO REJECT

x2: Reject H0 (p < 0.001) → coefficient ต่างกันอย่างมีนัยสำคัญ → heterogeneous effect

x3: Fail to reject H0 → coefficient ไม่ต่างกัน → homogeneous effect

5. QR for Count Data

Machado & Santos Silva (2005)

ปัญหา: QR กับข้อมูล Count

ปัญหา: QR ปกติออกแบบมาสำหรับ continuous data

Count data (เช่น จำนวนครั้งพบแพทย์) เป็น discrete → CDF มี jumps → quantile ไม่ unique!

วิธีแก้ — Jittering:

แปลง count y ให้เป็น continuous โดย:

z = y + u    โดย u ~ Uniform(0,1)

จากนั้นใช้ QR ปกติกับ z แล้ว transform กลับ

ลองดู: Jittering ทำให้ count data smooth

ขั้นตอน Quantile Count Regression

  1. Jitter: สร้าง z = y + u โดย u ~ U(0,1) → ทำให้ count data เป็น continuous
  2. QR: ประมาณ standard linear QR บน z → ได้ conditional quantiles ของ z
  3. Transform back: แปลงกลับ: Q̂q(y|x) = max(0, ⌈Q̂q(z|x)⌉ − 1)
    โดย ⌈·⌉ = ceiling function (ปัดขึ้น)
  4. Average: ทำซ้ำหลายรอบ (replications) แล้วเฉลี่ย เพื่อลด jittering noise

Bounded from below: เพราะ z = y + u โดย y ≥ 0 และ u ≥ 0 ดังนั้น Qq(z|x) ≥ q เสมอ → max(0, ...) ป้องกันค่าติดลบ

ใน Stata: ใช้ qcount (user-written) ซึ่งทำ jittering + QR + transform + average ให้อัตโนมัติ พร้อม qcount_mfx สำหรับ marginal effects

Median = 5 ครั้ง แต่มี right tail ยาว (max = 144 ครั้ง!)

Summary

สรุปบทเรียน

Limitations & Practical Considerations

Quantile Crossing

QR ประมาณแต่ละ quantile แยกกัน ทำให้เส้น quantile อาจ ตัดกัน ได้ เช่น Q̂.25(y|x) > Q̂.75(y|x) สำหรับบาง x

แก้ได้ด้วย rearrangement method หรือ simultaneous QR

Standard Errors

qreg ให้ SE แบบสูตร (เร็วแต่ไม่เสถียร)

bsqreg ให้ SE แบบ bootstrap (ช้ากว่าแต่ reliable กว่ามาก)

→ ใช้ bsqreg เสมอ (ดูหน้าถัดไป)

Interpretation

QR coefficient บอก effect ต่อ conditional quantile ไม่ใช่ effect ต่อคนที่ "อยู่ที่ quantile นั้น"

เช่น coef ที่ q=.90 ไม่ได้หมายถึง effect ต่อคนรวยที่สุด แต่หมายถึง effect ต่อ 90th percentile ของ y|x

Computational note: สำหรับ dataset ขนาดใหญ่ QR + bootstrap อาจใช้เวลานาน ใน Stata ให้ตั้ง set seed เพื่อ reproducibility และเริ่มจาก reps() น้อยๆ ก่อน (เช่น 100) แล้วค่อยเพิ่ม

ทำไมต้อง Bootstrap SE?

qreg: SE จากสูตร

ต้อง ประมาณว่ามีข้อมูลกี่จุดแถวๆ เส้น quantile

ถ้าข้อมูลบาง (เช่น ที่ q=0.05 หรือ q=0.95) การประมาณจุดนี้จะไม่เสถียร

ปัญหา: ที่หาง ข้อมูลน้อย → SE ผิดง่าย

bsqreg: SE จาก Bootstrap

สุ่ม sample ใหม่จากข้อมูลเดิม (with replacement) แล้วรัน QR ซ้ำ เช่น 400 รอบ

ได้ coefficient 400 ค่า → ดูว่า กระจายแค่ไหน → นั่นคือ SE

ไม่ต้องประมาณความหนาแน่น → ทำงานดีทุก quantile

ลองดู: Bootstrap สร้าง SE อย่างไร

กดปุ่มเพื่อเพิ่ม bootstrap samples แล้วดู histogram ของ coefficient ที่ได้

reps = 0, SE = ?

สังเกต: ยิ่งเพิ่ม reps histogram ยิ่งชัด → band ±1 SE (แดงอ่อน) คือช่วงที่ ~68% ของ bootstrap coefficients ตกอยู่ → นี่คือความไม่แน่นอนของ estimate ที่ bsqreg รายงาน
