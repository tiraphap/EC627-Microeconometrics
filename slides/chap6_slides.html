1. Introduction & Panel Data Basics

ข้อมูลติดตามบุคคลเดิมหลายช่วงเวลา — ทำไมถึงดีกว่า cross-section?

Panel Data คืออะไร?

Panel Data (Longitudinal Data) = การเก็บข้อมูลจากหน่วยเดิม (i = 1,...,N) ซ้ำหลายช่วงเวลา (t = 1,...,T)

เรียกว่า Short Panel เมื่อ N ใหญ่ T เล็ก (เช่น สำรวจ 595 คน 7 ปี)

ข้อดีหลัก: ควบคุม unobserved heterogeneity ที่คงที่ตามเวลา (time-invariant) ได้ — เช่น ความสามารถ พรสวรรค์ แรงจูงใจส่วนบุคคล

สิ่งที่ cross-section ทำไม่ได้เพราะ αi เหล่านี้สังเกตไม่ได้

ตัวอย่าง: ศึกษาผลของประสบการณ์ต่อค่าจ้าง — ถ้าใช้ cross-section อาจ confound กับ ability (คนเก่งมักได้ค่าจ้างสูงและทำงานนาน) Panel ช่วยแยก ability ออกได้

โครงสร้างข้อมูล Panel

Individual-Effects Model

yit = x'itβ + αi + εit

Fixed Effects (FE): อนุญาตให้ Corr(αi, xit) ≠ 0

αi เป็น parameter ที่ต้องประมาณ (หรือกำจัดออก)

ข้อเสีย: ประมาณค่าสัมประสิทธิ์ของตัวแปรที่คงที่ตามเวลา (เช่น เพศ การศึกษา) ไม่ได้

Random Effects (RE): สมมติ Corr(αi, xit) = 0

αi เป็น random variable จาก distribution บางอย่าง

ข้อดี: ประมาณค่าสัมประสิทธิ์ของตัวแปร time-invariant ได้

คำถามสำคัญ: αi สัมพันธ์กับ xit หรือไม่?

  • ถ้าสัมพันธ์ → ใช้ FE (consistent)
  • ถ้าไม่สัมพันธ์ → ใช้ RE (efficient กว่า)
  • ทดสอบด้วย Hausman test
คุณสมบัติFERE
Corr(αi, xit)อนุญาต= 0
Time-invariant varsไม่ได้ได้
Consistencyเสมอ*ถ้า H0 จริง
Efficiencyต่ำกว่าสูงกว่า (ถ้า H0)

*ภายใต้ strict exogeneity: E(εit|xi1,...,xiT,αi) = 0

2. Pooled OLS & PA Estimators

เริ่มจากวิธีง่ายที่สุด — แล้วทำไมมันยังไม่ดีพอ?

Pooled OLS

Pooled OLS = ใช้ OLS ธรรมดากับข้อมูล panel ทั้งหมด โดยไม่สนใจโครงสร้าง panel

ปัญหา: ละเลย αi ทำให้ error = αi + εit มี serial correlation สูงมาก

* Pooled OLS - cluster-robust SE regress lwage exp exp2 wks ed, vce(cluster id) * เปรียบเทียบกับ default SE (ผิด!) regress lwage exp exp2 wks ed

ปัญหา SE: Default SE เล็กเกินจริงมาก เพราะไม่คำนึงถึง within-person correlation → ต้องใช้ cluster-robust SE เสมอ!

Autocorrelation ใน residuals:

Lag 1: ρ ≈ 0.80 — สูงมาก! แสดงว่า αi ที่ไม่ได้ควบคุมสร้าง persistent correlation

* ตรวจ autocorrelation quietly regress lwage exp exp2 wks ed, vce(cluster id) predict uhat, residuals forvalues j = 1/6 { quietly corr uhat L`j'.uhat display "Lag `j' = " r(rho) }

ผลลัพธ์: Lag 1 = 0.80, Lag 2 = 0.72, ..., Lag 6 = 0.55

Autocorrelation ลดช้ามาก → เป็นเพราะ unobserved individual effect αi ที่คงที่ตลอดเวลา

Population-Averaged (PA) Estimator

PA estimator = Pooled FGLS ที่ model within-person correlation structure

ระบุ working correlation matrix (เช่น AR(2), exchangeable) แล้วใช้ GEE

ตอบคำถาม: "ค่าเฉลี่ยของ y เปลี่ยนอย่างไรเมื่อ x เปลี่ยน across the population?"

* PA estimator with AR(2) correlation xtreg lwage exp exp2 wks ed, pa corr(ar 2) vce(robust) nolog * ดู estimated correlation matrix matrix list e(R)

PA vs FE/RE:

  • PA ไม่แยก αi ออกจาก model — รวมเข้า error structure แทน
  • ให้ marginal effects โดยตรง (averaged over αi)
  • Robust SE ทำให้ valid แม้ correlation structure ผิด

เมื่อไหร่ใช้ PA?

  • สนใจ population-level effect ไม่ใช่ individual-level
  • Nonlinear model ที่ FE มีปัญหา incidental parameters
  • ต้องการ robust inference โดยไม่ต้อง assume αi distribution

3. Within (Fixed Effects) Estimator

กำจัด unobserved heterogeneity ด้วย mean-differencing

Within Transformation

แนวคิด: หัก mean ของแต่ละคนออก เพื่อกำจัด αi

yit - ȳi = (xit - x̄i)'β + (εit - ε̄i)

ผลลัพธ์: αi หายไป! เพราะ αi - αi = 0

OLS บน transformed data ให้ consistent estimates แม้ Corr(αi, xit) ≠ 0

ข้อจำกัดสำคัญ: ตัวแปร time-invariant (เช่น ed) ถูกกำจัดไปด้วย!

เพราะ xit - x̄i = 0 สำหรับทุก t → ไม่สามารถประมาณ coefficient ได้

* Within (FE) estimator xtreg lwage exp exp2 wks ed, fe vce(cluster id) * สังเกต: ed ถูก drop เพราะ time-invariant * R-sq within = 0.1727 * R-sq between = 0.2369
* LSDV (Least Squares Dummy Variable) * ให้ผลเหมือน FE แต่รวม dummy ทุกคน areg lwage exp exp2 wks ed, absorb(id) vce(cluster id) * หรือใช้ factor variables regress lwage exp exp2 wks ed i.id, vce(cluster id)

LSDV vs xtreg, fe: ให้ β เหมือนกันทุกประการ LSDV แค่ประมาณ αi ทุกตัวด้วย (594 dummy variables!)

Within vs Between Variation

Within vs Between Variation Visualizer

Total variation แยกได้เป็น:

xit - x̄ = (xit - x̄i) + (x̄i - x̄)

Within + Between

FE ใช้เฉพาะ within variation — ดูว่า "เมื่อคนเดิมมีประสบการณ์เพิ่มขึ้น ค่าจ้างเปลี่ยนอย่างไร?"

BE ใช้เฉพาะ between variation — "คนที่มีประสบการณ์เฉลี่ยมากกว่า มีค่าจ้างเฉลี่ยสูงกว่าหรือไม่?"

สังเกตใน interactive: เมื่อสลับไป Within จะเห็นว่าแต่ละกลุ่ม (แต่ละคน) ถูก center ที่ 0 — ความแตกต่างระหว่างคนหายไป

First-Difference Estimator

First-Difference (FD): อีกวิธีกำจัด αi โดยหักค่าช่วงเวลาก่อนหน้า

Δyit = Δx'itβ + Δεit

FD vs FE:

  • ถ้า T = 2: FD = FE (ให้ผลเหมือนกันทุกประการ)
  • ถ้า T > 2: FE efficient กว่าถ้า εit เป็น iid; FD efficient กว่าถ้า εit เป็น random walk
  • FD ต้องการ sequential exogeneity ที่อ่อนกว่า strict exogeneity ของ FE
* First-differences estimator sort id t regress D.(lwage exp exp2 wks ed), vce(cluster id) noconstant * D. = first-difference operator * noconstant เพราะ D.ed = 0 เสมอ

ผลจาก PSID: FD ให้ coefficient ของ exp ใกล้เคียงกับ FE แต่ SE ต่างกันเล็กน้อย — เพราะ εit อาจไม่ใช่ iid

4. Between & Random Effects Estimators

ใช้ทั้ง within และ between variation ให้เป็นประโยชน์

Between (BE) Estimator

Between Estimator: OLS บน group means

ȳi = x̄'iβ + αi + ε̄i
  • ใช้ข้อมูลเพียง N observations (หนึ่งต่อคน)
  • ประมาณ coefficient ของตัวแปร time-invariant ได้ (เช่น ed)
  • ไม่ consistent ถ้า Corr(αi, x̄i) ≠ 0 (ซึ่งมักเป็นจริง)
  • ไม่ค่อยนิยมใช้ในทางปฏิบัติ แต่ช่วยเข้าใจ RE ได้
* Between estimator xtreg lwage exp exp2 wks ed, be * Robust SE ด้วย bootstrap xtreg lwage exp exp2 wks ed, be vce(boot, reps(400) seed(10101))

RE = weighted average ของ FE และ BE:

β̂RE = λ · β̂FE + (1-λ) · β̂BE

โดย λ ขึ้นอยู่กับ σ2ε / σ2α ratio

Random Effects (RE) Estimator

RE Model: สมมติ αi ~ iid(0, σ2α) และไม่สัมพันธ์กับ xit

yit - θȳi = (xit - θx̄i)'β + (1-θ)αi + (εit - θε̄i)

θ (theta) = 1 - √(σ2ε / (Tσ2α + σ2ε))

  • θ → 1: RE → FE (เมื่อ σ2α ใหญ่มาก)
  • θ → 0: RE → OLS (เมื่อ σ2α ≈ 0)
* RE estimator with theta reported xtreg lwage exp exp2 wks ed, re vce(cluster id) theta * คำนวณ theta ด้วยตัวเอง display 1 - sqrt(e(sigma_e)^2 / (7*e(sigma_u)^2+e(sigma_e)^2))

ผลจาก PSID:

  • σu = 0.327 (SD of αi)
  • σe = 0.231 (SD of εit)
  • ρ = 0.668 — 67% ของ variance มาจาก αi
  • θ ≈ 0.80 — RE ใกล้ FE มาก (เพราะ αi สำคัญ)
  • ed coefficient = 0.094 (มีนัยสำคัญ — RE ประมาณได้แต่ FE ไม่ได้)

5. Hausman Test & Comparison

FE หรือ RE? — ให้ข้อมูลตอบ

Hausman Test: หลักการ

H0: Corr(αi, xit) = 0 → RE consistent & efficient

H1: Corr(αi, xit) ≠ 0 → เฉพาะ FE consistent

H = (β̂FE - β̂RE)' [V̂FE - V̂RE]-1 (β̂FE - β̂RE) ~ χ2K

Logic: ถ้า H0 จริง ทั้ง FE และ RE consistent → β̂FE ≈ β̂RE → H เล็ก

ถ้า H1 จริง RE biased แต่ FE ยังดี → β̂FE ≠ β̂RE → H ใหญ่

* Standard Hausman test quietly xtreg lwage $xlist, fe estimates store FE quietly xtreg lwage $xlist, re estimates store RE hausman FE RE, sigmamore

ผล PSID: χ2(3) = 32.42, p < 0.001

Reject H0 → αi สัมพันธ์กับ xit → ควรใช้ FE

Robust Hausman (Wooldridge 2002): ใช้ auxiliary regression แทน — ทำงานได้ดีกว่าเมื่อ error ไม่ iid

ข้อควรระวัง Hausman test:

  • ต้อง fit FE ก่อน แล้ว RE — ลำดับสำคัญใน Stata
  • Standard version assume homoskedasticity — ใช้ robust version ดีกว่า
  • บางที VFE - VRE ไม่ positive definite → ใช้ sigmamore option

6. Nonlinear Panel Data Models

Binary, Tobit, Count — ปัญหาที่ linear model ไม่มี

Nonlinear Panel: ภาพรวม

แนวทางเดิม 4 แบบยังใช้ได้:

  • Pooled — ignore panel structure
  • PA (GEE) — model correlation, marginal effects
  • RE — assume αi independent of x
  • FE — allow correlation

แต่มีปัญหาใหม่!

  • Incidental Parameters Problem: FE ใน nonlinear model ไม่ consistent เมื่อ T fixed, N → ∞
  • ยกเว้น conditional logit & Poisson FE ที่มี sufficient statistics
  • Linear FE ไม่มีปัญหานี้เพราะ within-transform กำจัด αi ได้พอดี

Incidental Parameters Problem

Logit FE (T=3): bias ≈ 50%! ใน short panel ผลลัพธ์ FE logit อาจ inflate coefficients มากจนเชื่อถือไม่ได้

ข้อมูล: Rand Health Insurance Experiment

RAND HIE: การทดลอง RCT ที่สุ่มอัตราร่วมจ่ายประกัน (coinsurance rate) → ลด endogeneity

Unbalanced panel: บางคนอยู่ 1–5 ปี

use npanel.dta, clear xtset id year xtdescribe summarize dmdu med mdu lcoins ndisease female age lfam child

ตัวแปรตาม:

  • dmdu = binary (ไปพบแพทย์หรือไม่) → Logit
  • med = continuous, 22% เป็นศูนย์ → Tobit
  • mdu = count (จำนวนครั้งพบแพทย์) → Poisson/NegBin

Regressors:

  • lcoins = log(coinsurance rate) — randomly assigned!
  • ndisease = number of chronic diseases
  • female = เพศหญิง (time-invariant)
  • age = อายุ (time-varying)
  • lfam = log(family size)
  • child = เด็ก (< 18 ปี)

Time-varying regressors (age, lfam, child) มี within variation — FE ได้

Time-invariant (female, ndisease) — FE drop, ต้อง RE หรือ PA

แนะนำ: ถ้าสงสัย endogeneity ใช้ Poisson FE กับ robust SE — consistent, ไม่ต้อง assume distribution shape

Quiz: Hausman Test

คำถาม: Hausman test ให้ χ2 = 2.15 (p = 0.54) จากข้อมูลที่ค่อนข้างเล็ก (N=50, T=3) ควรสรุปอย่างไร?

A) ใช้ RE ได้อย่างมั่นใจเพราะ p-value สูง
B) ยังไม่ควรสรุปว่า RE ดีกว่า — Hausman test อาจมี low power ใน small samples, ควรพิจารณาทฤษฎีประกอบ
C) ใช้ FE ดีกว่าเสมอเพราะ consistent
D) ผลไม่มีความหมายเพราะ Hausman test ใช้กับ nonlinear model เท่านั้น

Quiz: Nonlinear Panel Models

คำถาม: นักวิจัยมี panel data (N=500, T=4) ที่ตัวแปรตามเป็นจำนวนครั้งที่ไปโรงพยาบาล (count) และมี overdispersion สูง ควรเลือก estimator ใด?

A) Negative Binomial FE — เพราะจัดการ overdispersion ได้ดีที่สุด
B) Poisson FE กับ panel-robust SE — consistent แม้ overdispersed และไม่มี incidental parameters problem
C) OLS กับ log(count+1) — เพราะง่ายกว่า nonlinear model
D) Tobit FE — เพราะมีค่า 0 จำนวนมาก
