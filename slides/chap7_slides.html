1. บทนำและแรงจูงใจ

RD Design: เครื่องมือทรงพลังเมื่อไม่สามารถทำ RCT

ทำไมต้อง Regression Discontinuity?

ปัญหาหลักของการอนุมาน Causal Inference

Randomized Controlled Trials (RCTs) เป็น gold standard แต่มีข้อจำกัดด้านจริยธรรม ค่าใช้จ่าย และความเป็นไปได้ในการดำเนินการ

RD Design ใช้ประโยชน์จาก "การทดลองธรรมชาติ"

  • การได้รับ treatment ถูกกำหนดโดย score (running variable) ข้าม cutoff ที่กำหนดไว้ล่วงหน้า
  • หน่วยที่อยู่ใกล้ cutoff ทั้งสองฝั่งมีลักษณะ เปรียบเทียบกันได้ (comparable)
  • ความต่างของผลลัพธ์ที่ cutoff = ผลกระทบเชิงสาเหตุ (causal effect)

ตัวอย่างในชีวิตจริง

ทุนการศึกษา (คะแนนสอบ > เกณฑ์), สิทธิ์ได้รับสวัสดิการ (รายได้ < เกณฑ์), นโยบาย Head Start (อัตราความยากจน > cutoff)

Ref: Cattaneo, Idrobo & Titiunik (2020, 2023) A Practical Introduction to Regression Discontinuity Designs

องค์ประกอบหลักของ RD Design

Running Variable (X)

  • Score ที่กำหนดการได้รับ treatment
  • ต้องเป็นตัวแปรต่อเนื่อง (continuous) หรือกึ่งต่อเนื่อง
  • เช่น คะแนนสอบ, อัตราความยากจน, อายุ

Cutoff (c)

  • จุดตัดที่กำหนดไว้ล่วงหน้า (predetermined)
  • หน่วยไม่สามารถ manipulate ได้อย่างแม่นยำ
  • เช่น คะแนนสอบ = 50, poverty rate = 59.1984

2. สองกรอบแนวคิดของ RD

Continuity-based vs Local Randomization

กรอบแนวคิดที่ 1: Continuity-Based

สมมติฐานหลัก

Potential outcomes ต่อเนื่อง ที่จุด cutoff

Sharp RD Treatment Effect

วิธีการประมาณค่า

  • Local polynomial regression รอบ cutoff
  • เลือก bandwidth อย่างเหมาะสม
  • ใช้ kernel weighting ให้น้ำหนักจุดใกล้ cutoff
  • Robust bias-corrected inference

กรอบแนวคิดที่ 2: Local Randomization

สมมติฐานหลัก

ภายในหน้าต่างแคบ ๆ รอบ cutoff การได้รับ treatment เสมือนถูกสุ่ม (as-if random) — potential outcomes เป็นอิสระจาก running variable ในหน้าต่างนั้น

ข้อดี

  • วิเคราะห์คล้าย RCT ภายใน window
  • ใช้ Fisherian inference (permutation-based)
  • เหมาะกับ discrete running variable
  • ทดสอบ balance ของ covariates ได้ตรงไปตรงมา

3. Sharp RD vs Fuzzy RD

Perfect compliance vs Imperfect compliance

Sharp RD Design

นิยาม

Treatment ถูกกำหนดอย่างสมบูรณ์โดย running variable:

  • เหนือ cutoff: ทุกหน่วยได้รับ treatment (T=1)
  • ใต้ cutoff: ไม่มีหน่วยใดได้รับ treatment (T=0)
  • ความน่าจะเป็นกระโดดจาก 0 เป็น 1 ที่ cutoff

Treatment Effect

ตัวอย่าง: Head Start

  • เมือง (county) ที่มีอัตราความยากจน > 59.1984 ได้รับเงินทุน Head Start
  • Running variable: poverty rate ปี 1960
  • Treatment: ได้รับเงินทุนโครงการ
* Sharp RD estimation gen R = $X - $C gen T = ($X >= $C) reg $Y T##c.R if abs(R)<=9, robust rdrobust $Y R, h(9) kernel(uni) vce(hc0)

Fuzzy RD Design

นิยาม

การข้าม cutoff มีอิทธิพลอย่างมาก ต่อ treatment แต่ไม่ได้กำหนดอย่างสมบูรณ์:

  • ความน่าจะเป็นกระโดด แต่ ไม่ใช่จาก 0 ไป 1
  • มี noncompliance: บางหน่วยเหนือ cutoff ไม่ได้รับ treatment

Fuzzy RD = IV ที่ cutoff

คล้าย Wald estimator / Local Average Treatment Effect (LATE) สำหรับ compliers ที่ cutoff

* Fuzzy RD estimation rdrobust $Y $X, c($C) fuzzy(treatment_var)

4. การประมาณค่าและ Inference

rdrobust, Bandwidth Selection, Kernel Choices

Local Polynomial Regression

หลักการ

ประมาณ regression function แยกกันทั้งสองฝั่งของ cutoff โดยใช้ polynomial ภายใน bandwidth h:

Kernel Weighting

  • Uniform: น้ำหนักเท่ากันทุกจุดใน bandwidth
  • Triangular: น้ำหนักมากที่ cutoff, ลดลงตามระยะ
  • Triangular เป็น MSE-optimal สำหรับ boundary estimation

Polynomial Order

  • p=1 (local linear): แนะนำเป็นค่าเริ่มต้น
  • p=2 (local quadratic): สำหรับ robustness check
  • p สูงเกินไป → overfitting, variance สูง

Bandwidth Selection

Bias-Variance Trade-off

  • Bandwidth แคบ: Bias ต่ำ แต่ Variance สูง (ข้อมูลน้อย)
  • Bandwidth กว้าง: Variance ต่ำ แต่ Bias สูง (รวมจุดไกล cutoff)

MSE-Optimal Bandwidth

เลือก h ที่ minimize Mean Squared Error = Bias² + Variance

* Bandwidth selection rdbwselect $Y $X, c($C) kernel(uni) all * MSE-optimal: mserd * CER-optimal: cerrd (shorter)

Robust Bias-Corrected Inference

rdrobust ใช้ bandwidth สองขั้นตอน:

  • h: bandwidth หลักสำหรับ point estimation
  • b: bandwidth สำหรับ bias correction (มักใหญ่กว่า h)
  • ให้ robust confidence intervals ที่คำนึงถึง bias

ผลลัพธ์สำคัญของ rdrobust

  • Conventional: point estimate + SE (อาจมี bias สูง)
  • Bias-corrected: point estimate ปรับ bias แล้ว
  • Robust: CI ที่ robust ต่อ bias → แนะนำใช้สำหรับ inference

5. ตัวอย่างเชิงประจักษ์: Head Start

Ludwig & Miller (2007, QJE)

Head Start Program

บริบท

  • Head Start: โครงการช่วยเหลือเด็กจากครอบครัวยากจนในสหรัฐอเมริกา

* === ขั้นตอนที่ 2: RD Plots === rdplot $Y $X, c($C) binselect(esmv) // IMSE-optimal bins rdplot $Y $X, c($C) p(1) // linear fit
* === ขั้นตอนที่ 3: Replicate Ludwig & Miller === reg $Y T##c.R if abs(R)<=9, robust // OLS with bandwidth=9 rdrobust $Y R, h(9) kernel(uni) vce(hc0) // rdrobust equivalent

6. Falsification & Diagnostics

การตรวจสอบความถูกต้องของ RD Design

ทำไมต้อง Falsification Tests?

RD Assumption ไม่สามารถทดสอบโดยตรง

เราไม่สามารถตรวจสอบได้โดยตรงว่า potential outcomes ต่อเนื่องที่ cutoff แต่สามารถทำการทดสอบที่ ถ้าล้มเหลว แสดงว่า RD design มีปัญหา

การทดสอบ ตรวจสอบอะไร Stata Command
Density Test ไม่มีการ manipulation ที่ cutoff rddensity
Covariate Balance ตัวแปร pre-treatment ไม่กระโดดที่ cutoff rdrobust (ใช้กับ covariates)
Placebo Outcomes ผลลัพธ์ที่ไม่ควรถูกกระทบไม่มี discontinuity rdrobust (ผลลัพธ์ปลอม)
Placebo Cutoffs ไม่มี discontinuity ที่ cutoff อื่น rdrobust (cutoff ปลอม)
Donut Hole ผลไม่ถูกขับเคลื่อนด้วยจุดใกล้ cutoff มาก rdrobust (ตัดจุดใกล้ cutoff)

Density Test: ตรวจสอบ Manipulation

McCrary / Cattaneo et al. Density Test

  • ถ้าหน่วยสามารถ manipulate running variable → จะเห็นการกระจุกตัว (bunching) ที่ฝั่งใดฝั่งหนึ่งของ cutoff
  • H₀: density ต่อเนื่องที่ cutoff
  • ปฏิเสธ H₀ → ระวัง! อาจมี sorting
rddensity $X, c($C) * p-value > 0.05 → ดี (ไม่มี manipulation)

* Placebo cutoff ที่ R=2 และ R=3 rdrobust $Y R if R>0, c(2) kernel(uni) vce(hc0) rdrobust $Y R if R>0, c(3) kernel(uni) vce(hc0)

ตัดข้อมูลใกล้ cutoff มาก ๆ ออก:

7. สรุปและแนวทางปฏิบัติ

Practical Recommendations for RD Analysis

แนวทางปฏิบัติที่ดี

1. เริ่มต้นด้วยกราฟ

  • RD Plot เป็นสิ่งแรกที่ต้องทำ
  • ดู discontinuity ด้วยตาก่อน
  • ใช้ rdplot กับ binselect(esmv)
  • ลองทั้ง linear และ polynomial fit

2. Localize การวิเคราะห์

  • ใช้ local polynomial ไม่ใช่ global
  • เลือก bandwidth อย่างเป็นระบบ
  • ทดสอบ sensitivity กับ h ต่าง ๆ
  • ใช้ robust inference

3. Falsification เข้มงวด

  • Density test ทุกครั้ง
  • Balance test บน covariates
  • Placebo outcomes & cutoffs
  • Donut hole robustness

4. เลือกกรอบแนวคิดให้เหมาะสม

Continuity-based สำหรับ running variable ต่อเนื่อง | Local randomization สำหรับ discrete scores หรือต้องการ RCT-like inference

สรุป RD Checklist

  1. ระบุ running variable, cutoff, treatment
  2. สร้าง RD Plot (rdplot)
  3. Density test (rddensity)
  4. Balance test บน pre-treatment covariates

การประมาณค่า

  1. rdrobust (MSE-optimal BW)
  2. รายงาน robust CI
  3. ทดสอบ sensitivity กับ bandwidth
  4. เพิ่ม covariates (efficiency gain)

หลังการประมาณค่า

  1. Placebo outcomes
  2. Placebo cutoffs
  3. Donut hole robustness
  4. ลอง polynomial orders ต่าง ๆ
  5. ลอง kernel ต่าง ๆ

  • ทั้ง conventional & robust estimates
