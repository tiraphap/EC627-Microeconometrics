Potential Outcomes, SUTVA, และ Conditional Independence

  • = ผลลัพธ์เมื่อไม่ได้รับ treatment
  • = ผลลัพธ์เมื่อได้รับ treatment

Treatment Effect สำหรับหน่วย i

Fundamental Problem of Causal Inference

เราสังเกตได้เพียง ผลลัพธ์เดียว สำหรับแต่ละหน่วย:

โดย = สถานะการรับ treatment

ตัวอย่าง

ถ้า เราเห็น แต่ไม่เห็น (counterfactual) ดังนั้น ไม่สามารถคำนวณได้โดยตรง

พารามิเตอร์เชิงสาเหตุ: ATE และ ATT

Average Treatment Effect (ATE)

= ผลกระทบเฉลี่ยสำหรับทุกคนในประชากร

ถามว่า: "ถ้าบังคับทุกคนรับ treatment ผลจะเป็นอย่างไร?"

Average Treatment Effect on the Treated (ATT)

= ผลกระทบเฉลี่ยสำหรับผู้ที่ได้รับ treatment จริงเท่านั้น

ถามว่า: "คนที่เลือกรับ treatment ได้ประโยชน์แค่ไหน?"

DID ให้ ATT — เพราะเปรียบเทียบกลุ่มที่ได้รับนโยบายจริง (treated) กับกลุ่มควบคุม ไม่ได้ประมาณผลสำหรับทุกคนในประชากร

ATE vs ATT: ตัวอย่างเปรียบเทียบ

ตัวอย่างที่ 1: ค่าแรงขั้นต่ำ

ATE: ถ้าบังคับทุกจังหวัดขึ้นค่าแรง → การจ้างงานเฉลี่ยทั้งประเทศเปลี่ยนเท่าไหร่?

ATT: เฉพาะจังหวัดที่ขึ้นค่าแรงจริง → การจ้างงานเปลี่ยนเท่าไหร่?

ตัวอย่างที่ 2: โครงการอบรมอาชีพ

ATE: ถ้าส่งทุกคนตกงานไปอบรม → รายได้เฉลี่ยเพิ่มเท่าไหร่?

ATT: เฉพาะคนที่สมัครเข้าอบรมจริง → รายได้เพิ่มเท่าไหร่?

ทำไม ATE ≠ ATT?

Selection bias: คนที่เลือกรับ treatment มักมีลักษณะต่างจากคนทั่วไป

ค่าแรง: จังหวัดที่ขึ้นค่าแรงมักเป็นจังหวัดอุตสาหกรรม ตลาดแรงงานตึงตัว → ผลกระทบอาจน้อยกว่าจังหวัดเกษตรที่ยังไม่ขึ้น

อบรม: คนที่สมัครมักมีแรงจูงใจสูง → ได้ประโยชน์มากกว่าคนที่ถูกบังคับเข้าอบรม

SUTVA: Stable Unit Treatment Value Assumption

สมมติฐานพื้นฐาน

SUTVA ประกอบด้วย 2 ส่วน:

1. No Interference

ผลลัพธ์ของหน่วย i ไม่ขึ้นกับสถานะ treatment ของหน่วยอื่น

ตัวอย่างการละเมิด: วัคซีนสร้างภูมิคุ้มกันหมู่ ทำให้คนไม่ฉีดก็ได้ประโยชน์ (spillover)

2. No Hidden Variation in Treatment

ไม่มีรูปแบบ treatment ที่ซ่อนอยู่ หน่วยที่ได้รับ treatment ทุกหน่วยได้รับ treatment แบบเดียวกัน

ตัวอย่างการละเมิด: นโยบายค่าแรงขั้นต่ำที่แต่ละรัฐบังคับใช้เข้มงวดไม่เท่ากัน

Conditional Independence Assumption (CIA)

CIA คืออะไร?

เมื่อควบคุม X แล้ว → การได้รับ treatment (D) ไม่เกี่ยวกับ potential outcomes

พูดง่ายๆ: ในกลุ่มคนที่มีลักษณะ X เหมือนกัน ใครได้/ไม่ได้ treatment เสมือนถูกสุ่ม

Confounders (X) คืออะไร?

ตัวแปรที่มีผลต่อ ทั้ง D (treatment) และ Y (outcome)

ถ้าไม่ควบคุม X → เปรียบเทียบ treatment กับ control ไม่ fair → bias

CIA: ตัวอย่าง

ตัวอย่าง: ผลของปริญญาโทต่อรายได้

D = เรียนจบ ป.โท • Y = รายได้ • คำถาม: ป.โท ทำให้รายได้สูงขึ้นจริงหรือ?

ถ้าไม่ควบคุม X

คนเรียน ป.โท อาจมี ability สูงกว่าอยู่แล้ว → รายได้สูงไม่ใช่เพราะ ป.โท แต่เพราะเก่ง

X = ability เป็น confounder → ทำให้ ป.โท ดูมีผลมากกว่าความจริง

ถ้า CIA จริง

เปรียบเทียบคนที่ ability เท่ากัน (X เท่ากัน) → คนนึงเรียน ป.โท คนนึงไม่เรียน

ผลต่างรายได้ = causal effect ของ ป.โท จริงๆ

= ATE (ถ้า CIA จริง)

ปัญหา: CIA มักไม่สมจริง!

Unobserved Confounders

ในทางปฏิบัติ เรามักควบคุม X ได้ไม่ครบ — มีตัวแปรที่ วัดไม่ได้ แต่มีผลต่อทั้ง D และ Y

กลับมาที่ตัวอย่าง ป.โท

เราควบคุม ability ได้ (เช่น จาก GPA, คะแนนสอบ) แต่ยังมีสิ่งที่วัดไม่ได้:

  • แรงจูงใจ — คนที่อยากเรียนมากมักทำงานหนักกว่า
  • Connections — มหาวิทยาลัยดีมี network ที่ช่วยหางาน
  • ฐานะครอบครัว — ครอบครัวรวยส่งเรียนได้ + มี safety net
  • Signaling — ป.โท อาจแค่ส่งสัญญาณ ไม่ได้เพิ่ม productivity

ทางออก: DID

แทนที่จะพยายามควบคุม X ให้ครบ → ใช้ ข้อมูลก่อน-หลัง นโยบาย → กำจัด confounders ที่คงที่ตามเวลา (time-invariant) ออกไปได้ทั้งหมด — ไม่ต้องวัดมันเลย

2. DID: แนวคิดพื้นฐาน

สมการ DID, การตั้ง 2x2 Table, และการตีความ

แนวคิดหลักของ Difference-in-Differences

ไอเดียหลัก

ใช้ 2 กลุ่ม (treatment vs. control) และ 2 ช่วงเวลา (ก่อน vs. หลัง) เพื่อกำจัด confounders ที่คงที่ตามเวลา

Setup พื้นฐาน

  • 2 ช่วงเวลา: t = 0 (ก่อน) และ t = 1 (หลัง)
  • 2 กลุ่ม: Treatment (Group = 1) และ Control (Group = 0)
  • ที่ t = 0 ไม่มีใครได้รับ treatment
  • ที่ t = 1 เฉพาะ treatment group ได้รับ treatment

3. Common Trends Assumption

สมมติฐานที่สำคัญที่สุดของ DID

สมมติฐาน Common (Parallel) Trends

สมมติฐานหลัก

ถ้าไม่มี treatment ทั้ง 2 กลุ่มจะมี แนวโน้มการเปลี่ยนแปลงที่เหมือนกัน

สิ่งที่ต้องเป็นจริง

  • ระดับ (level) ต่างกันได้
  • แต่ การเปลี่ยนแปลง (trend) ต้องเหมือนกัน
  • ถ้าไม่มีนโยบาย ทั้งสองกลุ่มจะขึ้น/ลงเท่ากัน

ถ้าสมมติฐานผิด (Violation)

  • Treatment group มี trend ที่ต่างจาก control อยู่แล้ว
  • DID estimate จะมี bias
  • Bias = ส่วนต่างของ trend ที่ไม่ได้มาจาก treatment

การทดสอบ (ไม่สมบูรณ์)

ดู pre-treatment trends: ถ้าก่อน treatment ทั้ง 2 กลุ่มมี trend ขนานกัน ก็พอมั่นใจได้ (แต่ไม่ได้พิสูจน์) ว่าหลัง treatment ก็จะขนานถ้าไม่มี treatment

Common Trends Violation: เกิดอะไรขึ้น?

Common Trends สมมติว่า

ถ้า ไม่มี treatment ทั้งสองกลุ่มจะเปลี่ยนแปลงในอัตราเดียวกัน

พูดง่ายๆ: trend ของ treatment group (ถ้าไม่ได้รับ treatment) = trend ของ control group

ถ้าสมมติฐานนี้ผิด?

สมมติ trend ของ treatment group สูงกว่า control อยู่แล้ว:

DID จะ ประมาณค่า δ สูงเกินจริง เพราะรวม trend ที่ต่างกันเข้าไปด้วย

ตัวอย่าง: ค่าแรงขั้นต่ำ

ถ้า NJ มีเศรษฐกิจโตเร็วกว่า PA อยู่แล้ว (ไม่เกี่ยวกับค่าแรง):

  • การจ้างงาน NJ จะเพิ่มขึ้นมากกว่า PA แม้ไม่มีนโยบาย
  • DID จะคิดว่าส่วนที่เพิ่มคือผลของค่าแรงขั้นต่ำ
  • แต่จริงๆ แค่ NJ โตเร็วกว่า → overestimate δ

วิธีตรวจสอบ

  • Pre-trends test: ดูข้อมูลหลายปีก่อนนโยบาย trend ขนานกันมั้ย?
  • Event study: plot สำหรับ k < 0 ควร ≈ 0
  • Placebo DID: ลองทำ DID กับช่วงก่อนนโยบาย ไม่ควรพบ effect

สมการ Two-way Fixed Effects (TWFE)

ส่วนประกอบ

Event Study Design

แนวคิด

แทนที่จะประมาณ เดียว ให้ประมาณ treatment effect แยกตามระยะเวลาก่อน/หลัง treatment

Pre-trends Test

  • สำหรับ k < 0 ควร ไม่แตกต่างจาก 0
  • ถ้า pre-treatment coefficients = 0 แสดงว่า parallel trends น่าเชื่อ
  • ถ้า pre-treatment coefficients มี trend ชัดเจน แสดงว่า parallel trends อาจถูกละเมิด

Post-treatment Effects

  • สำหรับ k > 0 แสดง dynamic treatment effects
  • เห็นว่า treatment effect เพิ่มขึ้น/คงที่/ลดลงตามเวลา
  • k = -1 เป็น reference period (normalized to 0)

Autor (2003): Employment-at-Will Doctrine และ Temporary Employment

คำถามวิจัย

การยกเว้นหลัก employment-at-will มีผลต่อการจ้างงานแรงงานชั่วคราว (Temporary Help Services: THS) หรือไม่?

Employment-at-Will Doctrine

หลักกฎหมายแรงงานสหรัฐ: นายจ้างสามารถเลิกจ้างลูกจ้างได้ทุกเมื่อ โดยไม่ต้องมีเหตุผล

แต่ระหว่างปี 1979-1995 รัฐต่างๆ เริ่มยอมรับ ข้อยกเว้น ที่จำกัดสิทธิ์นี้:

  • mico - Implied Contract Exception
  • mppa - Public Policy Exception
  • mgfa - Good Faith Exception

สมมติฐาน

เมื่อการเลิกจ้างยากขึ้น (ข้อยกเว้น employment-at-will) นายจ้างจะหันไปใช้ แรงงานชั่วคราวแทน เพราะ:

  • ยืดหยุ่นกว่า
  • ไม่ต้องรับผิดชอบกฎหมายเลิกจ้าง
  • ลดความเสี่ยงทางกฎหมาย

DID Design

Treatment: รัฐที่ยอมรับข้อยกเว้น
Control: รัฐที่ยังไม่ยอมรับ
Outcome: log THS employment share

ปัญหา

Key Takeaways

สิ่งที่ต้องจำ

  1. DID = ผลต่างของผลต่าง กำจัด time-invariant confounders และ common shocks
  2. Parallel Trends คือสมมติฐานที่สำคัญที่สุด ต้องทดสอบด้วย pre-trends
  3. TWFE ขยาย DID ไปหลายช่วงเวลาและกลุ่ม ด้วย group FE + time FE
  4. Robustness: เพิ่ม state-specific trends, placebo tests, event study

Autor (2003) สอนอะไร?

  • DID ใช้กับ staggered policy adoption ได้ดี
  • ต้องตรวจสอบ robustness เสมอ
  • ผลที่ไม่ robust (mppa, mgfa) อาจมาจาก differential trends
  • Implied contract (mico) เป็นข้อยกเว้นเดียวที่มีผล robust ต่อ THS
